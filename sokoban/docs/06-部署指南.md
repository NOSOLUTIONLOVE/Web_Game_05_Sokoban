## 二、方案 A：Vercel 部署（推荐）

### A.1 通过 Vercel 网站（最简单）

**前置**：GitHub 账号 + Vercel 账号（用 GitHub 登录即可）

#### 步骤

1. **推送代码到 GitHub**

   ```bash
   cd /Users/carlshen/Documents/Project/Web_Game_01
   git add .
   git commit -m "feat: sokoban v0.1.0 - 完整可玩 + 20 关 + 84 个测试"
   git push origin main
   ```

2. **在 Vercel 导入项目**
   - 访问 https://vercel.com/new
   - 选择 **Import Git Repository**
   - 找到 `Web_Game_01` 仓库，点击 **Import**

3. **配置项目**

   Vercel 会自动识别为 Vite 项目。如需调整：

   | 配置项 | 值 |
   | --- | --- |
   | Framework Preset | Vite |
   | Build Command | `cd games/sokoban && npm run build` |
   | Output Directory | `games/sokoban/dist` |
   | Install Command | `cd games/sokoban && npm install` |
   | Root Directory | `games/sokoban`（推荐，子项目部署） |

   **注意**：如果仓库里同时有 snake / 2048 / tetris / flappybird / sokoban 等多个项目，每个项目要单独配置 Root Directory。

4. **环境变量（可选）**

   本项目无需环境变量。

5. **点击 Deploy**

   Vercel 会：
   - 拉取代码
   - `npm install`
   - `npm run build`（产物写入 `dist/`）
   - 部署到全球 CDN
   - 分配 URL：`https://web-game-01-sokoban.vercel.app`

6. **后续更新**

   ```bash
   git push origin main  # 自动触发重新部署
   ```

### A.2 通过 Vercel CLI（高级）

```bash
npm install -g vercel
cd games/sokoban
vercel              # 首次部署，交互式登录
vercel --prod       # 部署到生产环境
```

## 三、方案 B：Cloudflare Pages

### B.1 步骤

1. 推送代码到 GitHub（同上）
2. 访问 https://pages.cloudflare.com/
3. **Create a project** → 选 GitHub 仓库
4. 配置：
   - **Build command**: `cd games/sokoban && npm run build`
   - **Build output directory**: `games/sokoban/dist`
   - **Root directory**: `games/sokoban`
5. 点击 **Save and Deploy**

### B.2 优劣

- ✅ 免费层慷慨
- ✅ 全球 CDN
- ❌ 不像 Vercel 那样与 Vite 深度集成

## 四、方案 C：GitHub Pages

### C.1 步骤

1. 仓库根创建 `.github/workflows/deploy.yml`：

   ```yaml
   name: Deploy Sokoban
   on:
     push:
       branches: [main]

   jobs:
     deploy:
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: games/sokoban
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: 'npm'
             cache-dependency-path: games/sokoban/package-lock.json
         - run: npm ci
         - run: npm run build
         - uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: games/sokoban/dist
   ```

2. **Settings → Pages → Source**: 选 `gh-pages` 分支

### C.2 优劣

- ✅ 0 成本（与 GitHub 仓库绑定）
- ❌ 无全球 CDN（GitHub 自有 CDN 较慢）
- ❌ 配置较繁琐

## 六、配置详解

### 6.1 Vercel 配置（`vercel.json`）

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**作用**：所有路径请求都重写到 `index.html`，让 React Router（或纯 hash 路由）正常工作。

### 6.2 相对路径

`vite.config.ts` 中：

```ts
export default defineConfig({
  base: './',  // 相对路径，部署到任意子路径都可工作
});
```

这样打包后的 `index.html` 使用相对路径引用资源，部署到 Vercel 子路径（`https://xxx.vercel.app/sokoban/`）也能正常工作。

### 6.3 localStorage 隔离

每个域有独立的 localStorage。如果部署到 `vercel.app` 子域名，多个项目不冲突。

## 十、常见问题

### Q1: 部署后页面空白

**原因**：路径错误。

**解决**：
- 确认 `vite.config.ts` 中 `base: './'`
- 确认 `vercel.json` 中重写规则正确
- 浏览器 DevTools → Network 检查 404 资源

### Q2: 音效不播放

**原因**：iOS Safari 需要用户交互后才能播放 AudioContext。

**解决**：本项目已在 `Input.handleAction` 中调用 `audio.resume()`，确保首次用户点击"开始游戏"后音频可用。

### Q3: 移动端触屏不响应

**原因**：可能未正确阻止默认行为。

**解决**：
- `<canvas>` 设置 `touch-action: none`
- 触屏事件 `passive: true`（已实现）
- 滑动阈值 30px（已实现）

### Q4: 测试在 CI 中失败

**原因**：缺少 `happy-dom` 依赖（仅 GameEngine 测试需要）。

**解决**：本项目已在 `package.json` devDependencies 中包含 `happy-dom`。

### Q5: localStorage 不生效

**原因**：
- 隐私模式下 localStorage 抛异常（已 try-catch 吞掉）
- 域名 / 协议不同

**解决**：本项目 `lib/storage.ts` 已做异常吞咽处理。

### Q6: 通关后进度丢失

**原因**：可能是隐私模式 / 清缓存。

**解决**：进度持久化在 `sokoban:store` localStorage 键中。

## 十三、CDN 缓存策略

Vercel 自动配置：
- `dist/**` → `Cache-Control: public, max-age=31536000, immutable`
- `index.html` → `Cache-Control: public, max-age=0, must-revalidate`

这样 HTML 总拉新版本，静态资源永久缓存（hash 文件名）。

## 十四、安全注意事项

### 14.1 数据安全

- ✅ 仅 localStorage（无后端）
- ✅ 无第三方 SDK 上报（默认）
- ✅ 无 cookie / tracking

### 14.2 XSS 防护

- ✅ React 默认转义所有文本
- ✅ 不使用 `dangerouslySetInnerHTML`

### 14.3 CSP（可选）

在 `vercel.json` 添加：

```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Content-Security-Policy",
          "value": "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'"
        }
      ]
    }
  ]
}
```
