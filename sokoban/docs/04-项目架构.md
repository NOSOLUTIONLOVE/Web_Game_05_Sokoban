# 04 - 项目架构

> 推箱子三层架构 + 数据流 + 文件清单

---

## 一、整体架构

采用 **三层分离** 架构：

```
┌────────────────────────────────────────┐
│  UI 层（React + Zustand）              │
│  - HUD / Menu / Overlays / ActionBar  │
│  - 订阅 store 状态变化                  │
│  - 触发用户操作（点击 / 键盘 / 触屏）   │
└──────────────┬─────────────────────────┘
               │ 回调（单向数据流）
┌──────────────▼─────────────────────────┐
│  状态层（Zustand + persist）           │
│  - useGameStore                        │
│  - 持久化关卡进度 + 音量设置            │
└──────────────┬─────────────────────────┘
               │ getState().action()
┌──────────────▼─────────────────────────┐
│  引擎层（纯 TypeScript）               │
│  - GameEngine / Board / UndoStack     │
│  - DeadlockDetector / LevelManager    │
│  - Renderer / Input                    │
│  - 状态私有持有，仅事件触发回调         │
└────────────────────────────────────────┘
```

## 二、核心数据流

### 2.1 启动流程

```
main.tsx
  ↓
App.tsx
  ↓
SokobanGame.tsx (useEffect)
  ↓
  创建 GameEngine（持有 Board / UndoStack / LevelManager）
  ↓
  创建 Renderer（绑定 canvas）
  ↓
  注册引擎回调（onPhaseChange / onMove / onWin / onDeadlock...）
  ↓
  初始化 store 状态
  ↓
  启动 requestAnimationFrame 渲染循环
  ↓
  启动 Input 系统（绑定 window + container）
```

### 2.2 用户操作流程

```
用户按 ↑ 键
  ↓
Input.handleKey('arrowup')
  ↓
cb.onAction('moveUp')
  ↓
SokobanGame.handleAction('moveUp')
  ↓
engine.handleMove('up')
  ↓
Board.tryMove('up')
  ↓
返回 MoveResult
  ↓
GameEngine 内部状态变更
  ↓
触发回调：onMove / onPush / onSettle / onBlocked / onWin
  ↓
回调更新 store
  ↓
React 组件订阅 store 变化，自动 re-render
  ↓
Renderer 下一帧渲染新状态
```

### 2.3 状态机

```
        startGame           togglePause
menu ─────────────► playing ◄──────────► paused
  ▲                  │   ▲                  │
  │                  │   │                  │
  │   backToMenu     │   │ togglePause      │ backToMenu
  │   resetLevel     │   │                  │
  └──────────────────┘   │                  │
                         │ handleWin        │
                         ▼                  │
                        over ◄──────────────┘
                         │
                         │ nextLevel / backToMenu
                         ▼
                       playing / menu
```

## 四、关键设计模式

### 4.1 引擎内部状态私有

```ts
export class GameEngine {
  private levelManager: LevelManager;
  private board: Board;
  private undoStack: UndoStack;
  private _phase: GamePhase = 'menu';
  // ...
}
```

外部只能通过 getter（currentLevel / currentBoard / currentProgress）查询，不能修改。

### 4.2 单向数据流

```
用户操作 → Engine → 回调 → Store → UI re-render
```

- UI 触发动作（dispatch）
- Engine 修改内部状态
- Engine 触发回调
- 回调更新 Store
- Store 通知 React 组件 re-render

### 4.3 事件订阅

```ts
new GameEngine({
  onPhaseChange: (p) => store.setPhase(p),
  onMove: (m, p) => store.setMoveCount(m, p, ...),
  onWin: (m, p, opt) => store.setFlashWin(...),
  // ...
});
```

事件是单向的：Engine → Store，无反向依赖。

### 4.4 Context Provider 暴露 Engine

```ts
const EngineContext = createContext<GameEngine | null>(null);

export function useEngine(): GameEngine {
  const engine = useContext(EngineContext);
  if (!engine) throw new Error('useEngine must be used within SokobanGame');
  return engine;
}
```

子组件可通过 `useEngine()` 获取引擎实例。

### 4.5 渲染快照模式

```ts
getRenderSnapshot(): RenderSnapshot {
  return {
    board: this.board,
    moveCount: this._moveCount,
    pushCount: this._pushCount,
    phase: this._phase,
    isOptimal: ...,
    // ...
  };
}
```

Renderer 每帧拉取快照渲染，避免双向耦合。

## 五、状态管理

### 5.1 持久化（localStorage）

| 键名 | 内容 | 写入时机 |
| --- | --- | --- |
| `sokoban:store` | unlockedLevels / bestMoves / bestPushes / cleared / audioEnabled | Zustand persist 自动 |
| `sokoban:progress` | LevelProgress（兼容旧版） | 通关时 |

### 5.2 内存状态

- 引擎状态：cells / boxes / player / undoStack（私有，浏览器内存）
- 渲染状态：settleAnims / blockedAnim / winAnim（私有，Renderer 持有）
- React 状态：组件局部 UI 状态（如 LevelSelect 的 open）
