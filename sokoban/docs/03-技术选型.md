## 二、关键决策

### 2.1 React vs Vue

✅ **选 React**

- 与项目库其他游戏（snake / 2048 / tetris / flappybird）保持一致
- 生态成熟（shadcn/ui / Framer Motion / Zustand）
- 团队熟悉度高

### 2.2 Canvas 2D vs SVG / DOM

✅ **选 Canvas 2D**

- 网格类游戏（推箱子）元素数量动态（最多 200 元素），Canvas 性能稳定
- 像素级控制（动效 / 渐变 / 阴影）
- 60fps 流畅

### 2.3 Zustand vs Redux

✅ **选 Zustand**

- 极简 API（无 reducer / action / dispatcher）
- 体积小（1KB）
- persist 中间件开箱即用
- 与项目库其他游戏一致

### 2.4 Tailwind vs CSS-in-JS

✅ **选 Tailwind**

- 原子化，开发快
- 编译期生成 CSS，体积小
- 与项目库其他游戏一致

### 2.5 XSB 内置 vs 关卡编辑器

✅ **选内置 20 关（XSB 格式）**

- Microban 风格经过验证
- XSB 是行业标准，参考 sokobano.de
- 关卡编辑器复杂度高，V1 不做

### 2.6 撤销栈：栈式 vs 增量式

✅ **选栈式（snapshot 模式）**

- 实现简单
- 200 步历史内存可控
- 撤销 / 重做天然支持

### 2.7 死锁检测：保守 vs 完整

✅ **选保守（角落 + 边墙）**

- 覆盖 90% 常见情况
- 实现简单，性能好
- 漏判时通过 toast 提示用户手动撤销

### 2.8 音效：素材 vs Web Audio API

✅ **选 Web Audio API 实时合成**

- 零素材依赖（避免版权问题）
- 包体积小
- 5 个音效实现简单
