## 1. 项目概述

### 1.1 背景
推箱子（Sokoban）诞生于 1982 年，是经典的解谜游戏品类。玩家扮演仓库工人推箱子到指定位置，涉及关卡设计、撤销/回放、死锁检测等核心算法。是"非动作类游戏"开发的代表案例。

### 1.2 目标
- **学习目标**：掌握关卡数据格式（XSB）、状态撤销、死锁检测（可选）、解谜算法。
- **产品目标**：提供 20+ 内置关卡，含编辑器与撤销回放。

### 1.3 范围
**包含**：20+ 内置关卡、撤销/重做、关卡选择、移动步数统计。
**不包含**：在线关卡编辑器、用户关卡分享、联机对战。

## 3. 核心玩法

### 3.1 玩法描述
在二维网格地图上，玩家（仓库工人）只能上下左右移动一格。如果移动方向上是一个箱子，工人会推动箱子向同方向移动一格（前提是该方向上没有墙或其他箱子）。所有箱子都推到目标点上时关卡通过。

### 3.2 元素类型

| 元素 | 符号 | 说明 |
| --- | --- | --- |
| 墙 | `#` | 不可通过，不可推 |
| 地板 | ` ` (空格) | 工人可站立 |
| 目标点 | `.` | 地板，箱子到此即视为归位 |
| 箱子 | `$` | 可推动 |
| 归位箱子 | `*` | 箱子已在目标点上 |
| 工人 | `@` | 玩家角色 |
| 工人在目标点上 | `+` | 工人站在目标点上 |

### 3.3 计分规则
- 无分数概念。
- 统计指标：步数、推箱数、时间、是否最优解（步数最少）。

### 6.2 视觉规范
- **网格**：每格 64×64 px，可根据关卡大小自适应缩放。
- **配色**：
  - 墙 `#3a3a3a`（深灰带砖纹）
  - 地板 `#8b6f47`（木地板色）
  - 目标点 `#fbbf24`（黄色脚印/十字）
  - 箱子 `#a16207`（棕色木箱）
  - 归位箱子 `#22c55e`（绿色木箱）
  - 工人 `#3b82f6`（蓝色小人）
- **字体**：标题用衬线体，模拟 80 年代游戏风格。

### 6.3 动效
- 工人移动：每格 100ms 平滑过渡。
- 箱子归位时：箱体闪烁绿光 2 次 + 缩放反弹。
- 推不动时：工人抖动一下表示失败。

## 8. 数据结构关键设计

### 8.1 关卡（Level）
```ts
type CellType = 'wall' | 'floor' | 'target' | 'box' | 'box_on_target' | 'player' | 'player_on_target';

interface Level {
  id: number;
  name: string;
  width: number;
  height: number;
  cells: CellType[][];
  optimalMoves?: number;
  optimalPushes?: number;
}
```

### 8.2 XSB 关卡文本格式
```xsb
####
# .#
#  ###
#*@  #
#  $ #
#  ###
####
```

### 8.3 状态（GameState）
```ts
interface GameState {
  level: Level;
  player: Point;
  boxes: Point[];
  targets: Point[];
  moveCount: number;
  pushCount: number;
  history: MoveRecord[];   // 用于撤销
  redoStack: MoveRecord[];
}
```

### 8.4 推箱算法
```ts
function tryMove(direction: Direction): boolean {
  const next = getNextPosition(player, direction);

  // 撞墙
  if (isWall(next)) return false;

  // 推动箱子
  if (hasBox(next)) {
    const beyond = getNextPosition(next, direction);
    if (isWall(beyond) || hasBox(beyond)) return false;
    moveBox(next, beyond);  // 推箱子
  }

  player = next;
  return true;
}
```

## 9. 技术约束

- **渲染**：Canvas 2D（主网格 + UI 层）。
- **关卡格式**：XSB（Sokoban 行业标准格式）。
- **算法**：BFS/A* 可用于自动求解（V2）。
- **存储**：localStorage（关卡进度、撤销栈）。
- **构建**：Vite + TypeScript。
